# Guía de Contribución | Backup Tecno

¡Gracias por tu interés en contribuir al proyecto frontend de Backup Tecno!

## Proceso de Desarrollo

1. **Fork** el repositorio
2. **Crea una rama** para tu feature (`git checkout -b feature/AmazingFeature`)
3. **Commit** tus cambios (`git commit -m 'Add some AmazingFeature'`)
4. **Push** a la rama (`git push origin feature/AmazingFeature`)
5. **Abre un Pull Request**

## Estándares de Código

- Utiliza **TypeScript** para tipado estático
- Sigue las reglas de **ESLint**
- Usa **Tailwind CSS** para estilos
- Escribe componentes **funcionales** con hooks
- Documenta tus componentes con **comentarios JSDoc**

## Ejecución en Local

```bash
npm install
npm run dev
```

## Compilación y Testing

```bash
npm run build
npm run lint
```

## Estructura de Commits

```
<tipo>: <descripción corta>

<descripción más detallada si es necesario>
```

Tipos válidos:
- `feat:` - Nueva funcionalidad
- `fix:` - Corrección de bug
- `docs:` - Cambios en documentación
- `style:` - Cambios de formato/estilos
- `refactor:` - Refactorización de código
- `test:` - Cambios en tests

## Preguntas o dudas

Abre un **Issue** describiendo claramente tu pregunta.
